# セッション引き継ぎノート生成

セッション終了時や作業の区切りで、引き継ぎノートを生成する。

## 手順

1. 今回のセッションで行ったことを振り返る
2. 引き継ぎノートの出力先ディレクトリ（`.claude/handovers/`）の「基準ディレクトリ」を決める
3. 基準ディレクトリ配下に `.claude/handovers/` ディレクトリが存在しない場合は作成する
4. `YYYY-MM-DD_HHmm.md` のファイル名で引き継ぎノートを生成する（例: `2026-02-17_1430.md`）
5. 既存のファイルと同名になる場合は末尾に `_2` などの連番を付与する

## 出力先ディレクトリの決定ルール（git worktree 対応）

基本は「現在の git リポジトリのルート（`git rev-parse --show-toplevel`）」を基準ディレクトリとする。

ただし、`git worktree` で以下のような構成になっている場合は、worktree群の共通親ディレクトリに集約する：

例:

~/work/project/
  main/                # main / develop を置く “常用” worktree
  wt/
    feat-login/        # featureブランチ用 worktree
    fix-timeout/
    review-1234/

このとき、どの worktree (`main/` や `wt/*/`) で実行していても
**`~/work/project/.claude/handovers/`** に作成する。

### 判定ルール

- `git rev-parse --show-toplevel` の末尾が `/main` の場合  
  → 基準ディレクトリ = その親ディレクトリ（= `.../project`）

- `git rev-parse --show-toplevel` のパス中に `/wt/` を含む場合（例: `.../project/wt/feat-login`）  
  → 基準ディレクトリ = `/wt/` より手前（= `.../project`）

- 上記に当てはまらない場合  
  → 基準ディレクトリ = `git rev-parse --show-toplevel`

※結果として、出力先は常に  
`<基準ディレクトリ>/.claude/handovers/` になる。

## 引き継ぎノートの構成

必ず以下のセクションを含める。該当なしのセクションは「なし」と記載する。

### 作業ブランチ

- 作業を行ったブランチ

### 今回やったこと

- 作業内容と進捗を箇条書きで

### 決定事項

- 確定した設計判断、方針、ルールなど

### 捨てた選択肢と理由

- 検討したが採用しなかったアプローチとその理由

### ハマりどころ

- 詰まったポイント、エラー、想定外の挙動など

### 学び

- 今回得られた知見、気づき

### 次にやること

- 未完了のタスク、次のセッションで着手すべきこと
- 優先度がわかるように書く

### 関連ファイル

- 今回触った主要ファイルのパス一覧

## ルール

- 簡潔に書く
- 箇条書き中心
- 推測や曖昧な表現は避け、事実ベースで書く
- 「捨てた選択肢と理由」は特に重要。次のセッションで同じ議論を繰り返さないために
